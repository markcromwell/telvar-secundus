# Itch.io Release Procedure

This document outlines the standard procedure for releasing new builds of *Telvar Secundus* to Itch.io.

## 1. Download Release Artifact

All release builds are generated by the Continuous Integration (CI) pipeline on GitHub. The final HTML5 build is packaged into a zip file named `itch-package.zip`.

To download the artifact:
1. Navigate to the "Actions" tab in the project's GitHub repository.
2. Select the workflow run that corresponds to the commit you want to release (e.g., a run on the `main` branch).
3. On the summary page for the workflow run, scroll down to the "Artifacts" section.
4. Click on `itch-package` to download the `itch-package.zip` file.
5. Unzip the downloaded file to a temporary directory.

## 2. Upload to Itch.io

We use the `butler` command-line tool to upload builds to Itch.io.

1.  **Install Butler:** If you don't have it, follow the [official installation instructions](https://itch.io/docs/butler/installing.html).
2.  **Authenticate:** Log in to your Itch.io account using `butler login`. You only need to do this once per machine.
3.  **Push the build:** Navigate to the directory where you unzipped the artifact and run the following command. Replace `your-account` with the appropriate Itch.io account name.

    ```sh
    butler push . your-account/telvar-secundus:html5
    ```

This command pushes the contents of the current directory to the `html5` channel of the `telvar-secundus` project.

## 3. Verification Checklist

After the upload is complete, perform the following checks to ensure the release is valid.

- [ ] **Verify Zip Contents:** The `itch-package.zip` artifact should contain the `index.html`, `project.js`, `project.wasm`, and `project.pck` files. No source code or `.gd` files should be present.
- [ ] **Confirm Launch:** The build should launch and be playable directly in the browser on the Itch.io project page. The main menu should appear, and the game should start correctly.
- [ ] **Validate Timestamp:** The build page on Itch.io should show an updated timestamp corresponding to the new upload.

## Notes

### Credential Errors

If `butler` cannot find valid Itch.io credentials (e.g., `BUTLER_API_KEY` environment variable is not set or the local credential file is missing/invalid), it will fail gracefully.

- An explicit error message will be printed to the console indicating that authentication failed.
- The upload process will terminate immediately.
- No partial or corrupt build will be uploaded to Itch.io.
- To resolve, run `butler login` again and re-run the `butler push` command.
